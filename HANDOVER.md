# DB-CRM — 운영 인수인계 가이드

**DB-CRM (Customer & Data Management)** 관리자·담당자용 사용 매뉴얼 및 일상 운영 안내.

> 기술 배포·DB 설정은 [README.md](./README.md)를, 전체 제품 요구사항은 [prd.md](./prd.md)를 참고하세요.

---

## 빠른 시작

1. 브라우저에서 운영 URL 접속
2. 담당자ID + 비밀번호로 로그인
3. 상단 네비게이션 (또는 모바일에서 햄버거 메뉴)으로 기능 이동

### 역할별 보이는 메뉴 · 동작

> DB-CRM 은 **3단계 역할** 을 지원합니다 — `담당자 (agent)`, `매니저 (manager)`, `관리자 (admin)`. 매니저는 담당자 권한에 "전체 고객 조회 + 담당자 변경" 만 추가된 중간 계급입니다.

| 메뉴 / 동작 | 담당자 (권한 없음) | 담당자 (권한 부여) | 매니저 (manager) | 관리자 (admin) |
|---|:---:|:---:|:---:|:---:|
| 고객 목록 | ✓ (본인분) | ✓ (본인분) | ✓ (전체) | ✓ (전체) |
| 고객 검색 (이름·주소·전화·통화결과 등) | ✓ | ✓ | ✓ | ✓ |
| **담당자 필터** 드롭다운 | — | — | ✓ | ✓ |
| 테이블 컬럼 정렬·순서·폭 조절 | ✓ | ✓ | ✓ | ✓ |
| 팝업 상세 **열기** | ✓ (본인분) | ✓ (본인분) | ✓ (전체) | ✓ (전체) |
| 팝업 **방문주소 · 메모 · 통화결과 · 예약일시** 편집 | ✅ 항상 | ✅ | ✅ | ✅ |
| 팝업 **나머지 필드** 전체 편집 | — | ✓ (`canEdit`) | ✓ (`canEdit`) | ✓ |
| **담당자 필드 직접 변경** (팝업 드롭다운) | — | — | ✅ 기본 | ✅ |
| **고객 삭제** | — | ✓ (`canDelete`) | ✓ (`canDelete`) | ✓ |
| **엑셀 다운로드** | — | ✓ (`canExport`) | ✓ (`canExport`) | ✓ |
| **팝업 이미지 저장** | — | ✓ (`canDownloadImage`) | ✓ (`canDownloadImage`) | ✓ |
| 전화 걸기 · 복사 | ✓ | ✓ | ✓ | ✓ |
| **담당자 일괄 변경** (체크박스 → BulkBar) | — | — | ✅ 기본 | ✅ |
| **DB 등록일 일괄 변경** | — | — | — | ✓ |
| **엑셀 업로드** (`/admin/excel`) | — | — | — | ✓ |
| 사용자 관리 · 변경 이력 · 로그인 이력 | — | — | — | ✓ |
| 강제 로그아웃 | — | — | — | ✓ (본인 제외) |

> 💡 매니저는 본인의 `agentId` 로도 **고객을 할당받을 수 있어** 영업을 겸직할 수 있습니다.

## 고객 목록 · 검색

상단 검색바에서 복합 조건 검색 가능 (모두 AND):

- **이름** (부분 일치)
- **주소** (부분 일치)
- **전화번호** (하이픈 무시 substring — 예: `833` → `010-8330-xxxx`, `010-0833-xxxx` 등 매칭)
- **통화결과** (드롭다운)
- **담당자** (관리자만 선택 가능)
- **주민 앞 6자리** / **주민 뒤 7자리** (정확 일치)
- **출생연도 범위** (`1960`~`1970` 식)

**자동 검색**: 입력만 하면 0.4초 후 자동 반영 (검색 버튼 없음). 드롭다운은 즉시 반영. 검색 중엔 작은 스피너가 표시됩니다. **초기화** 버튼으로 모든 필터 해제.

표 제목 우측 **엑셀 다운로드** (`canExport` 보유 사용자만) — 현재 검색 조건 그대로 xlsx 추출.

통화결과 배지 색상: 예약=청록 · 부재=회색 · 가망=초록 · 거절=빨강 · 결번=회색 · 민원=노랑.

표에는 **엑셀 28컬럼 전체**가 표시됩니다 (기본 정렬: DB 등록일 내림차순).

> 💻 **울트라와이드(21:9) 자동 지원**: 화면 폭이 넓으면 28컬럼이 한 화면에 전부 펼쳐져 가로 스크롤 없이 볼 수 있습니다. 좁은 화면에서는 기존처럼 가로 스크롤이 유지됩니다.

### 테이블 개인화 (각 사용자·기기별)

- **정렬**: 헤더 텍스트 클릭 → 오름차순 ↔ 내림차순 토글. 화살표 아이콘으로 현재 방향 표시. 정렬 상태는 **URL 쿼리**에 저장되어 새로고침·공유 시 유지.
- **컬럼 순서 변경**: 헤더를 잡고 옆 컬럼으로 **드래그** → 위치 교환. 브라우저 localStorage에 저장되어 **다음 접속 시에도 유지**.
- **컬럼 폭 조절**: 헤더 오른쪽 경계(1~2px 라인, 커서가 ↔로 변함)를 **드래그** → 인접 두 컬럼이 서로 폭을 교환 (전체 표 폭은 변하지 않음). localStorage에 저장.
- **초기화**: 테이블 우상단 "컬럼 초기화" 버튼 → 기본 순서·폭으로 복원.

> 💡 각 사용자의 설정은 브라우저별·기기별로 개별 저장됩니다. 같은 사람이 다른 PC에서 로그인하면 해당 PC의 설정을 사용합니다.

## 팝업 상세 편집

목록에서 고객 이름 클릭 → 팝업 오버레이. 이전/다음 버튼으로 검색 결과 내 이동 (팝업 프레임은 그대로, 내부 데이터만 교체).

### 편집 가능 필드 — 권한별 차이

| 필드 | 담당자 (권한 없음) | 담당자 (`canEdit`) | 관리자 |
|---|:---:|:---:|:---:|
| **방문주소 · 메모 · 통화결과 · 예약일시** | ✅ | ✅ | ✅ |
| 이름 · 연락처 · 원주소 · 직업 | 🔒 | ✅ | ✅ |
| 생년월일 · 주민번호 뒤7 | 🔒 | ✅ | ✅ |
| 지사 · 본부 · 소속팀 | 🔒 | ✅ | ✅ |
| 보험사 · 상품명 · 가입일 · DB 만기일 · 예약일시 | 🔒 | ✅ | ✅ |
| DB 보험료 · 소분류 | 🔒 | ✅ | ✅ |
| 담당자 변경 (드롭다운) | — | — | ✅ |

> 🔒 = 진한 색 텍스트로 **읽기만 가능**, 배경만 옅은 회색으로 표시

### 연락처 전화 걸기 · 복사

연락처 입력창 우측 두 버튼 — **누구나** 사용 가능:

- **전화**: `tel:` URI로 OS 기본 전화 핸들러 호출 (Windows Phone Link, iOS FaceTime, 모바일 기본 전화앱)
- **복사**: 한국식 `010-0000-0000` 포맷으로 클립보드 복사

### 주민번호

평문으로 저장·표시됩니다. 팝업에는 **뒷자리 7자리**(`N + 6자리`) 입력창만 표시되며, **앞자리 6자리는 `생년월일`에서 자동 파생**(`YYMMDD`)되어 별도 입력 필요 없음. 엑셀 업로드로 이미 앞자리가 채워진 고객은 그대로 보존됩니다.

### 메모 — 우클릭으로 일시·담당자 자동 삽입

메모 textarea에서 **마우스 오른쪽 버튼**을 누르면 현재 위치에 다음 형식으로 자동 입력:

```
2026.04.19 11:51 (현재로그인사용자) :
```

직전 줄과 자동 줄바꿈으로 분리되며, 콜론 뒤로 커서가 이동해 바로 본문 작성 가능. 상담 기록 누적 시 손쉽게 타임스탬프 추가.

### 키보드 단축키

| 키 | 동작 |
|---|---|
| `Esc` | 팝업 닫기 |
| `Ctrl+S` | 저장 |
| `Ctrl+←` | 이전 고객 |
| `Ctrl+→` | 다음 고객 |

### 팝업 위치 이동 (데스크톱)

팝업 상단 툴바의 **빈 공간**(버튼 사이)을 마우스로 끌면 팝업을 자유롭게 이동할 수 있습니다. 백드롭은 투명해 뒤 화면이 보이며 외부 클릭으로 닫기는 그대로 동작.

### 이미지 저장

팝업 우상단 **이미지 저장** 버튼 (관리자 또는 `canDownloadImage` 권한 보유자만 노출) → 파일명 `{고객명}{생년2자리}{간단주소}.png` 로 다운로드.

예: `김동환79충북청주.png`

- 생년 2자리 = 생년월일 연도의 뒤 두 자리 (1979 → `79`)
- 간단주소 = 첫 두 토큰에서 `시·군·구·특별시·광역시` 접미사 제거 후 결합 (`충북 청주시 흥덕구 …` → `충북청주`)

- 편집 중인 입력값도 캡처에 반영됩니다.
- **팝업 전체 내용** 이 캡처되어, 스크롤로 아래에 가려진 부분까지 한 장의 세로로 긴 PNG 로 저장됩니다.
- **지사·본부·소속팀** 은 이미지에서 제외됩니다 (고객에게 전달할 때 불필요한 조직 정보 가리기).

## 엑셀 업/다운로드

### 업로드 (대량 이관) — `canCreate` 보유 사용자

1. **엑셀 업/다운로드** 메뉴 → 파일 드롭/선택
2. **미리보기** 클릭 → 파싱 결과 확인 (총 건수, 오류, 미등록 담당자)
3. 문제 없으면 **업로드 실행** → 확인 다이얼로그 → DB 반영

**중복 처리**:
- `고객코드No` 일치 → 기존 레코드 갱신
- 코드가 없으면 **(이름+전화)** 로 매칭해서 갱신
- 둘 다 없으면 신규 추가

**미등록 담당자 경고**: 엑셀의 `담당자ID`가 사용자 DB에 없으면 해당 행은 미배정 상태로 저장됩니다. 먼저 사용자 관리에서 계정을 추가해두는 것을 권장.

**주민번호 처리**: `주민No` 컬럼은 `YYMMDD-NXXXXXX` 형식 평문 그대로 저장됩니다.

### 다운로드 — `canExport` 보유 사용자

- **전체 다운로드**: `엑셀 업/다운로드` → **전체 다운로드**
- **검색 결과만**: 고객 목록에서 필터 적용 후 우상단 **엑셀 다운로드** 버튼
- 컬럼 28개 그대로 출력 (주민번호 평문 포함)
- 파일명: `DB-CRM_고객명부_YYYYMMDD-HHmm.xlsx`

## 사용자 관리 (관리자)

### 추가

1. 사용자 관리 → **사용자 추가**
2. 입력 항목:
   - **담당자ID** (예: `a00020`)
   - **이름**
   - **역할** — 셋 중 하나 선택:
     - `담당자 (agent)` — 본인 배정 고객만. 권한 5종은 관리자가 부여.
     - `매니저 (manager)` — 전체 조회 + 담당자 변경 **기본 부여**. 나머지 5종은 agent 와 동일한 방식으로 선택 부여.
     - `관리자 (admin)` — 모든 권한 자동 부여.
   - **초기 비밀번호** (6자 이상)
   - **담당자 권한 체크박스 5종** (담당자·매니저일 때 사용):
     - ☐ `입력 (신규 등록)` → 엑셀 업로드 페이지·API 사용
     - ☐ `수정` → 본인 배정 고객(매니저는 전체 고객)의 모든 필드 편집
     - ☐ `삭제` → 본인 배정 고객(매니저는 전체 고객) 삭제
     - ☐ `엑셀 다운로드` → xlsx 다운로드
     - ☐ `이미지 다운로드` → 고객 팝업의 [이미지 저장] 버튼 사용
   - 역할을 관리자로 선택하면 위 5개가 자동으로 **전체 체크 + 비활성** (관리자는 항상 모든 권한)
3. 생성 후 초기 비밀번호를 본인에게 전달

> 💡 담당자·매니저가 아무 권한도 없어도 **방문주소 · 메모 · 통화결과 · 예약일시**는 항상 편집 가능 (영업 현장 기본 수요 — 통화 중 즉석에서 예약을 잡을 수 있어야 함). 매니저는 추가로 **담당자 재할당(개별·일괄)** 도 플래그와 무관하게 가능.

### 수정

- 담당자ID는 생성 후 **변경 불가** (모든 고객 데이터의 FK)
- 이름 · 역할 · 권한 5종 체크박스는 언제든 수정 가능
- 본인 계정을 `담당자`로 강등 불가 (관리자 보호)
- 목록 테이블에 담당자별 활성 권한이 배지로 요약 표시됨:
  - 매니저: `전체조회` · `담당자변경` (기본 부여, 항상 ON) + `입력` / `수정` / `삭제` / `엑셀` / `이미지`
  - 담당자: `입력` / `수정` / `삭제` / `엑셀` / `이미지`
  - 관리자: `전체 권한` 단일 표시

### 비밀번호 재설정

관리자가 새 비밀번호를 직접 설정 → bcrypt 해시로 교체. 원래 비밀번호는 확인 불가.

### 삭제

- 본인 삭제 불가
- 담당 고객이 있는 경우 경고 표시 → 삭제 시 해당 고객들은 **미배정**으로 전환
- 삭제 후 동일 담당자ID로 재등록해도 과거 로그인 이력은 복원되지 않음

## 일괄 변경 (관리자)

고객 목록에서 체크박스로 여러 고객 선택 (헤더 체크박스로 현재 페이지 전체 선택) → 화면 하단에 **N건 선택됨** 플로팅 바 표시. 다음 두 가지 일괄 작업 가능 (모두 `bulk_change` 감사로그 기록):

### 담당자 일괄 변경

1. 플로팅 바 → **담당자 일괄 변경** 클릭
2. 새 담당자 선택(미배정 가능) → **N건 변경**

### DB 등록일 일괄 변경

1. 플로팅 바 → **DB 등록일 일괄 변경** 클릭
2. 새 날짜 선택 (또는 "DB 등록일을 비우기" 체크 → `null` 처리)
3. **N건 변경** → 일괄 적용

> ⚠ 되돌릴 수 없는 작업이지만 변경 이력에 건별 이전 값이 기록돼 수동 복원은 가능합니다.

## 고객 삭제 (관리자 또는 `canDelete` 담당자)

1. 고객 상세 팝업 우상단 **삭제** 버튼 (권한 없으면 숨김)
2. 확인 다이얼로그: "이 작업은 되돌릴 수 없습니다" 경고
3. 삭제 확정 시:
   - 고객 레코드 영구 삭제 (주민번호·주소·메모 등 모두)
   - **변경 이력에는 삭제 스냅샷이 남음** (이름·연락처·이전 담당자) — 복원은 수동으로만 가능
4. 실수 방지: 삭제 전 엑셀 다운로드로 백업 권장

## 변경 이력 (관리자)

관리자 → **변경 이력** 페이지. 고객 데이터에 대한 변경을 추적합니다.

### 표시 정보

- 일시, 작업자, 액션 배지, 대상 고객(링크), 변경 내용 요약
- 액션 종류:
  - `편집` (파란색): 일반 편집·삭제
  - `담당자 변경` (청록): 개별 고객의 담당자 수정
  - `일괄 변경` (보라): 다중 고객 담당자 또는 DB 등록일 일괄 변경
  - `주민번호 조회` (빨강): 과거 복호화 모달이 있던 시기의 레거시 액션 (현재 신규 발생 없음)

### 감사 스냅샷에 포함되는 필드

팝업 상세 편집(`edit` · `agent_change`) 시 `before` / `after` 에 기록되는 필드:

- **기본정보**: 이름 · 생년월일 · 연락처 · 직업 · 원주소 · 방문주소 · 주민번호 앞/뒷자리
- **DB 정보**: 보험사 · 보험상품명 · DB 보험료 · 소분류 · 가입일 · DB 만기일 · DB 등록일
- **조직**: 지사 · 본부 · 소속팀
- **상태**: 통화결과 · 예약일시 · 메모 · 담당자ID

일괄 변경(`bulk_change`):
- **담당자 일괄 변경**: `{ agentId: 이전 → 새값 }` 만 기록
- **DB 등록일 일괄 변경**: `{ dbRegisteredAt: 이전 → 새값 }` 만 기록

고객 삭제: `{ deleted: false, name, phone1, agentId } → { deleted: true }` 스냅샷.

### 필터

- 액션 종류
- 작업자 (담당자ID)
- 시작일 / 종료일

> 감사·분쟁 대응 시 이 기록을 증거로 활용할 수 있습니다.

## 로그인 이력 (관리자)

관리자 → **로그인 이력** 페이지. **인증 이벤트**(로그인 시도)를 기록합니다. (변경 이력과는 별개 — 고객 변경은 변경 이력, 로그인 시도는 로그인 이력)

### 표시 정보

- 일시 · 결과(성공 ✓ / 실패 ✗) · 담당자ID · 이름 · IP 주소 · 브라우저/OS · 실패 사유
- 실패 건은 연한 빨강 배경으로 강조

### 필터

- 사용자 (특정 담당자의 로그인만 조회)
- 결과 (성공만 / 실패만 / 전체)
- 시작일 / 종료일

### 활용

- 의심스러운 반복 실패(비번 유출 시도) 감지
- "누가 언제 어디서 로그인했는지" 증빙
- 퇴사자·휴직자 계정의 접근 감시

## 강제 로그아웃 (관리자)

특정 사용자의 모든 활성 세션을 즉시 종료시킵니다. (예: 퇴사자·계정 유출 의심 대응)

### 사용법

1. 관리자 → **사용자 관리**
2. 대상 행의 **"로그아웃"** 버튼 클릭 (본인 행은 비활성)
3. 확인 다이얼로그 → "강제 로그아웃"
4. 결과:
   - 대상 사용자가 **다음 페이지 이동 시 자동으로 `/login` 화면**으로 돌아감 (수초 내)
   - Slack / Telegram 에 알림 전송 (설정돼 있을 때)
5. 대상 사용자는 **정상적으로 다시 로그인 가능** (계정이 삭제·잠기는 게 아님)

> ⚠ 권한·데이터는 변경되지 않고 **세션만 무효화**됩니다. 영구 차단하려면 사용자 삭제 또는 비밀번호 재설정 후 본인에게 알리지 않으면 됩니다.

### 접속 상태 표시

사용자 관리 테이블 맨 왼쪽 컬럼:
- 🟢 **접속 중**: 최근 5분 내에 페이지 활동이 있음
- ⚪ **오프라인**: 5분 이상 활동 없음 또는 로그아웃 상태

> 판정 방식: Auth.js가 매 요청마다 `last_seen_at` 을 **1분마다** DB에 갱신. 5분 기준으로 접속 여부를 판정. 정확히 실시간은 아니고 최대 수 분의 오차가 있을 수 있습니다.

## 외부 알림 (Slack / Telegram)

관리자가 구성해두면 **로그인 이벤트가 실시간으로 외부 메신저**로 발송됩니다.

### 지원 채널

- **Slack**: Incoming Webhook URL 을 Vercel `SLACK_WEBHOOK_URL` 환경변수에 등록
- **Telegram**: BotFather로 봇 생성 → 토큰 + chat_id 를 `TELEGRAM_BOT_TOKEN` · `TELEGRAM_CHAT_ID` 에 등록

둘 다 설정하면 **양쪽 모두로 알림** 발송. 하나만 설정하면 그쪽만. 둘 다 미설정이면 알림은 꺼지고(다른 기능은 정상).

### 알림 이벤트

| 아이콘 | 이벤트 | 포함 정보 |
|---|---|---|
| 🔐 | 로그인 성공 | 이름·ID·역할·IP·브라우저·시각 |
| 🚨 | 로그인 실패 | 시도한 ID·IP·브라우저·사유·시각 |
| ⛔ | 강제 로그아웃 | 대상 사용자·실행 관리자·시각 |

### 운영 팁

- **Slack 알림 소리 안 옴?** 채널 설정 → 알림 → "모든 새 메시지" 로 변경 (기본값은 "멘션만"이라 봇 메시지가 무음 처리됨).
- **Telegram 이 더 공격적**: 모바일 앱이 기본적으로 팝업·소리·진동 제공.
- 민감한 정보를 받으므로 **웹훅 URL / 봇 토큰은 비밀**. 유출 시 즉시 Slack app 재생성 또는 Telegram BotFather `/revoke`.

## 보안 · 운영 설계 요약

| 항목 | 구현 |
|---|---|
| 비밀번호 | bcryptjs 12 round 해시 저장. 복원 불가. |
| 주민번호 | `rrn_front`(varchar 6) · `rrn_back`(varchar 7) **평문 저장**. 화면·엑셀 모두 평문 표시. |
| 세션 | JWT 기반, 8시간 만료. 유휴 30분 자동 로그아웃(5분 전 경고). |
| 권한 제어 | RBAC. 담당자는 `agent_id = session.agent_id` 조건이 쿼리에 강제 주입. 타인 고객 URL 직접 접근 시 차단. |
| 감사로그 | 편집·담당자변경·일괄변경 액션 자동 기록. 영구 보관. |
| 로그인 이력 | 성공·실패 전부 기록 (`login_events`). IP·UA·사유 포함. 영구 보관. |
| 강제 로그아웃 | `users.sessions_invalidated_at` 기반. JWT iat 비교로 세션 폐기. |
| 외부 알림 | Slack / Telegram 병행. 실패해도 로그인 자체는 정상 진행 (fire-and-forget). |
| 검색엔진 색인 | `robots: noindex` 메타 태그로 차단 (사내 시스템) |

### 키 관리 주의사항

- `AUTH_SECRET` 을 바꾸면 모든 사용자가 즉시 로그아웃됩니다.
- Vercel Environment Variables 에서만 관리하고 GitHub 에 커밋하지 않습니다.
- 주민번호는 평문이므로 **DB 접근 통제·전송 암호화(TLS)** 가 더욱 중요합니다. Neon 의 IP allow-list · 강력한 DB 비밀번호 관리 필수.

## 자주 발생하는 상황

### Q. 담당자가 비밀번호를 잊어버렸습니다.
A. 관리자 → 사용자 관리 → 해당 사용자 → **비밀번호 재설정**.

### Q. 엑셀에 있는 담당자 중 사용자 DB에 없는 계정이 있습니다.
A. 업로드 실행 전에 먼저 사용자 관리에서 계정을 추가하세요. 추가하지 않으면 해당 고객은 미배정 상태로 저장되고 나중에 일괄 변경으로 재배정해야 합니다.

### Q. 실수로 엑셀을 업로드해서 잘못된 담당자가 할당됐습니다.
A. 고객 목록에서 해당 고객들을 검색·선택 후 **담당자 일괄 변경**으로 올바른 담당자로 재할당하거나 미배정으로 전환. 변경 이력에 이전 담당자가 기록돼 있어 확인 가능.

### Q. 팝업에서 "다음"을 눌러도 같은 고객만 나옵니다.
A. 검색 결과가 1건이거나 현재 페이지 끝에 도달한 경우입니다. 페이지네이션으로 다음 페이지로 이동 후 재진입하세요.

### Q. 이미지 저장 파일에 한글 폰트가 깨져 나옵니다.
A. 브라우저가 Pretendard를 로드하지 못한 상태일 수 있습니다. 페이지 새로고침 후 다시 시도하세요.

### Q. 담당자가 고객 정보를 수정하려는데 대부분 필드가 흐려서 입력이 안 됩니다.
A. 해당 담당자에게 **`수정` 권한(`canEdit`)** 이 없어서입니다. 관리자 → 사용자 관리 → 해당 담당자 수정 → "수정" 체크박스를 켜주세요.
권한이 없어도 **방문주소 · 메모 · 통화결과 · 예약일시**는 상담 현장에서 필수라 항상 편집 가능합니다.

### Q. 담당자가 엑셀 업로드 메뉴에 들어갈 수가 없습니다.
A. **`입력` 권한(`canCreate`)** 이 필요합니다. 사용자 관리에서 체크해주세요.

### Q. 메모를 비웠는데 저장 후 원래 메모가 되살아납니다.
A. v1.3 에서 수정됐습니다. 발생 시 새로고침 후 재시도하고, 그래도 발생하면 개발팀에 문의.

### Q. 페이지 이동·팝업 열기가 느립니다.
A. 몇 가지 가능성:
- **평소 10초 이상 미사용 후 첫 클릭**: Neon DB Free tier 는 5분 idle 시 슬립 → 다음 요청에서 1~2초 깨우기 지연. 한두 번 클릭 후 안정화됨. Neon Pro 업그레이드로 해결 가능 ($19/월).
- **항상 느림**: Vercel 리전이 Singapore (`sin1`) 로 설정돼 있는지 확인 (Settings → Functions → Region). Neon 과 같은 리전이 아니면 쿼리마다 국제 왕복 지연 발생.
- **특정 사용자만 느림**: 사용자 네트워크 문제 가능. 다른 Wi‑Fi / LTE 로 접속 비교.
- v1.6 에서 Singapore 리전 이전 + React.cache 로 페이지당 DB 쿼리 수 감소 적용됨.

### Q. 전화 걸기 버튼을 눌렀는데 Phone Link가 안 뜹니다.
A. Windows 설정 → Apps → Default apps → Choose defaults by link type → **TEL** 을 **Phone Link**로 지정하세요. Phone Link와 휴대폰 페어링이 선행되어야 합니다.

### Q. 전화번호 검색에 "833"을 넣었는데 010-8330이 안 나옵니다.
A. 저장된 전화번호에 "833"이 substring 으로 포함돼 있어야 합니다. 하이픈은 자동 무시되지만 실제 숫자에 833 이 없으면 안 잡힙니다.

### Q. 검색 버튼이 사라졌어요.
A. 의도된 동작입니다. 입력만 하면 자동으로 검색됩니다 (텍스트 0.4초 디바운스, 드롭다운 즉시).

### Q. 팝업이 화면 가운데에 고정돼서 가려집니다.
A. 데스크톱에서는 팝업 상단 툴바의 빈 공간을 끌어 위치를 옮길 수 있습니다.

### Q. 모바일에서 메뉴가 안 보여요.
A. 화면 좌상단 **햄버거 아이콘 (☰)** 을 눌러주세요.

### Q. 테이블 컬럼 순서/폭을 바꿨는데 다른 PC에서 접속하면 원래대로 돌아갑니다.
A. 정상입니다. 개인화 설정은 **브라우저별 localStorage**에 저장되므로 PC·브라우저마다 별도입니다. 다시 맞추시거나 "컬럼 초기화" 후 사용하시면 됩니다.

### Q. Slack 채널에 알림이 안 와요.
A. 체크리스트:
1. Vercel 환경변수에 `SLACK_WEBHOOK_URL` 이 **정확히** 등록됐는지
2. 등록 후 **재배포**(Redeploy) 했는지
3. 해당 Slack 채널 알림 설정이 "모든 새 메시지"로 돼 있는지 (봇 메시지는 "멘션만"에선 무음)

### Q. Telegram 알림이 안 와요.
A. `TELEGRAM_BOT_TOKEN` 과 `TELEGRAM_CHAT_ID` **둘 다** 등록돼야 동작합니다. 하나만 등록하면 silent skip.

### Q. 퇴사자 계정을 잠시 차단하고 싶어요.
A. 두 가지 방법:
- **즉시 세션만 종료**: 강제 로그아웃 버튼 (대상은 다시 로그인 가능)
- **영구 차단**: 비밀번호 재설정(새 값 미공유) 또는 사용자 삭제(담당 고객 미배정 전환)

### Q. "접속 중" 표시가 실제와 다릅니다.
A. 최대 약 5분의 오차가 있습니다. DB 부하를 줄이려 `last_seen_at` 을 1분마다만 갱신하기 때문입니다. 즉시성이 필요한 경우엔 로그인 이력 페이지의 최근 로그인 시각을 참고하세요.

### Q. 고객을 실수로 삭제했습니다.
A. 삭제는 되돌릴 수 없습니다. **변경 이력**에서 마지막 스냅샷을 찾아 엑셀 업로드로 수동 복원하거나, 관리자가 직접 DB 수정(개발팀 요청) 필요합니다. **삭제 전 엑셀 다운로드 백업을 권장**합니다.
