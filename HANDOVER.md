# A-ONE 고객관리 시스템 — 운영 인수인계 가이드

에이원금융판매(주) 관리자·담당자용 사용 매뉴얼 및 일상 운영 안내.

> 기술 배포·DB 설정은 [README.md](./README.md)를, 전체 제품 요구사항은 [prd.md](./prd.md)를 참고하세요.

---

## 빠른 시작

1. 브라우저에서 배포 URL(예: `https://crm.aonefs.co.kr`) 접속
2. 담당자ID + 비밀번호로 로그인
3. 상단 네비게이션으로 기능 이동

### 역할별 보이는 메뉴

| 메뉴 | 담당자 (agent) | 관리자 (admin) |
|---|:---:|:---:|
| 고객 목록 | ✓ (본인 배정분만) | ✓ (전체) |
| 사용자 관리 | — | ✓ |
| 엑셀 업/다운로드 | — | ✓ |
| 변경 이력 | — | ✓ |
| 팝업 상세 편집 | ✓ | ✓ |
| 주민번호 등록 | ✓ | ✓ |
| 주민번호 복호화 열람 | — | ✓ |
| 담당자 일괄 변경 | — | ✓ |
| 팝업 이미지 저장 | ✓ | ✓ |

## 고객 목록 · 검색

- 상단 검색바에서 **이름**, **주소**(부분 일치), **통화결과**, **담당자**(관리자만), **주민 앞 6자리**, **주민 뒤 7자리** 조건 조합 검색
- 초기화 버튼으로 모든 필터 해제
- 표 헤더 우측 **엑셀 다운로드**(관리자) — 현재 검색 조건 그대로 xlsx 추출
- 통화결과 배지 색상: 예약=주황 · 부재=회색 · 가망=초록 · 거절=빨강 · 결번=회색 · 민원=노랑

## 팝업 상세 편집

목록에서 고객 이름 클릭 → 팝업 오버레이.

### 편집 가능 필드

**고객 정보(좌)**
- 이름 · 연락처 · 원주소 · 방문주소 · 직업
- 주민번호 앞 6 / 뒤 7 (입력 시 자동 암호화·해시 저장)

**보험 / 상담 정보(우)**
- 통화결과(예약/부재/가망/거절/결번/민원)
- 보험사 · 상품명 · 가입일 · 예약일시
- 담당자 변경 (관리자만)
- 메모

### 키보드 단축키

| 키 | 동작 |
|---|---|
| `Esc` | 팝업 닫기 |
| `Ctrl+S` | 저장 |
| `Ctrl+←` | 이전 고객 |
| `Ctrl+→` | 다음 고객 |

### 이미지 저장

팝업 우상단 **이미지 저장** 버튼 → `{고객명}_{YYYYMMDD-HHmm}.png` 다운로드. 편집 중인 값도 반영됩니다.

### 주민번호 복호화 열람 (관리자)

1. 고객 상세 → 주민번호 영역의 **복호화** 버튼
2. 열람 사유 입력 (선택) → **열람(기록됨)** 클릭
3. 전체 주민번호(`YYMMDD-1234567`) 표시 + 복사 버튼
4. 닫으면 메모리에서 즉시 폐기
5. **변경 이력**에 `주민번호 조회` 액션으로 자동 기록됨

> ⚠ 주민번호 열람은 개인정보보호법 관련 민감 작업입니다. 정당한 업무 목적이 있을 때만 사용하고, 사유를 구체적으로 적어주세요.

## 엑셀 업/다운로드 (관리자)

### 업로드 (대량 이관)

1. 관리자 > **엑셀 업/다운로드** → 파일 드롭/선택
2. **미리보기** 클릭 → 파싱 결과 확인 (총 건수, 오류, 미등록 담당자)
3. 문제 없으면 **업로드 실행** → 확인 다이얼로그 → DB 반영

**중복 처리**:
- `고객코드No` 일치 → 기존 레코드 **갱신**
- 코드가 없으면 **(이름+전화)** 로 매칭해서 **갱신**
- 둘 다 없으면 **신규 추가**

**미등록 담당자 경고**:
엑셀의 `담당자ID`가 사용자 DB에 없으면 해당 행은 담당자 **미배정** 상태로 저장됩니다. 먼저 사용자 관리에서 계정을 추가해두는 것을 권장합니다.

**주민번호 처리**:
업로드 시 `주민No` 컬럼(7자리)은 자동으로 **AES-256-GCM 암호화 + HMAC 해시**되어 저장됩니다. `생년월일`에서 YYMMDD 앞자리 해시도 동시에 계산되어 저장됩니다. DB에 평문은 저장되지 않습니다.

### 다운로드

- **전체 다운로드**: 관리자 > 엑셀 업/다운로드 > 전체 다운로드
- **검색 결과만**: 고객 목록에서 필터 적용 후 우상단 **엑셀 다운로드** 버튼
- 컬럼 28개 그대로 출력 (주민No는 마스킹)
- 파일명: `A-ONE_고객명부_YYYYMMDD-HHmm.xlsx`

## 사용자 관리 (관리자)

### 추가

1. 관리자 > 사용자 관리 > **사용자 추가**
2. 담당자ID (예: `a00020`), 이름, 역할(담당자/관리자), 초기 비밀번호, 소속(지사/본부/팀) 입력
3. 생성 후 초기 비밀번호를 본인에게 전달

### 수정

- 담당자ID는 생성 후 **변경 불가** (모든 고객 데이터의 FK)
- 이름·역할·소속은 언제든 수정 가능
- 본인 계정을 `담당자`로 강등 불가 (관리자 보호)

### 비밀번호 재설정

관리자가 새 비밀번호를 직접 설정 → bcrypt 해시로 교체. 원래 비밀번호는 확인 불가.

### 삭제

- 본인 삭제 불가
- 담당 고객이 있는 경우 경고 표시 → 삭제 시 해당 고객들은 **미배정**으로 전환
- 삭제 후 동일 담당자ID로 재등록해도 과거 로그인 이력은 복원되지 않음

## 담당자 일괄 변경 (관리자)

1. 고객 목록에서 체크박스로 여러 고객 선택 (헤더 체크박스로 현재 페이지 전체 선택)
2. 화면 하단에 **N건 선택됨** 플로팅 바 표시
3. **담당자 일괄 변경** 클릭 → 새 담당자 선택(미배정 가능)
4. **N건 변경** 클릭 → 일괄 적용 + 변경 이력에 각 건마다 `bulk_change` 기록

> ⚠ 되돌릴 수 없는 작업이지만 변경 이력에서 과거 담당자를 확인해 수동 복원은 가능합니다.

## 변경 이력 (관리자)

관리자 > **변경 이력** 페이지.

### 표시 정보

- 일시, 작업자, 액션 배지, 대상 고객(링크), 변경 내용 요약
- 액션 종류:
  - `편집` (파란색): 일반 편집
  - `담당자 변경` (주황): 개별 고객의 담당자 수정
  - `일괄 변경` (보라): 다중 고객 담당자 일괄 재배정
  - `주민번호 조회` (빨강): 관리자의 주민번호 복호화 열람

### 필터

- 액션 종류
- 작업자 (담당자ID)
- 시작일 / 종료일

> 감사·분쟁 대응 시 이 기록을 증거로 활용할 수 있습니다. 주민번호 조회 기록에는 열람 사유가 포함됩니다.

## 보안 · 개인정보 보호 설계 요약

| 항목 | 구현 |
|---|---|
| 비밀번호 | bcryptjs 12 round 해시 저장. 복원 불가. |
| 주민번호 뒷자리 | AES-256-GCM 암호화(표시용) + HMAC-SHA256 해시(검색용) 이중 저장. 키는 분리. |
| 주민번호 앞자리 | 생년월일 기반 YYMMDD의 HMAC 해시만 저장. |
| 세션 | JWT 기반, 8시간 만료. 유휴 30분 자동 로그아웃(5분 전 경고). |
| 권한 제어 | RBAC. 담당자는 `agent_id = session.agent_id` 조건이 쿼리에 강제 주입. 타인 고객 URL 직접 접근 시 404. |
| 감사로그 | 편집·담당자변경·일괄변경·주민번호조회 4종 액션 기록. 영구 보관. |
| 검색엔진 색인 | `robots: noindex` 메타 태그로 차단 (사내 시스템) |

### 키 관리 주의사항

- `PII_ENC_KEY`, `PII_HMAC_KEY`를 **한 번 배포한 뒤에는 절대 변경하지 않습니다**. 변경 시 기존 암호문·해시를 쓸 수 없게 됩니다.
- `AUTH_SECRET`을 바꾸면 모든 사용자가 즉시 로그아웃됩니다.
- Vercel Environment Variables에서만 관리하고 GitHub에 커밋하지 않습니다.
- 키를 분실하면 주민번호 복구가 불가능하므로 안전한 곳(비밀번호 관리자 등)에 별도 보관하세요.

## 자주 발생하는 상황

### Q. 담당자가 비밀번호를 잊어버렸습니다.
A. 관리자 > 사용자 관리 > 해당 사용자 > **비밀번호 재설정**.

### Q. 엑셀에 있는 담당자 중 사용자 DB에 없는 계정이 있습니다.
A. 업로드 실행 전에 먼저 사용자 관리에서 계정을 추가하세요. 추가하지 않으면 해당 고객은 미배정 상태로 저장되고 나중에 일괄 변경으로 재배정해야 합니다.

### Q. 실수로 엑셀을 업로드해서 잘못된 담당자가 할당됐습니다.
A. 고객 목록에서 해당 고객들을 검색·선택 후 **담당자 일괄 변경**으로 올바른 담당자로 재할당하거나 미배정으로 전환. 변경 이력에 이전 담당자가 기록돼 있어 확인 가능.

### Q. 팝업에서 "다음"을 눌러도 같은 고객만 나옵니다.
A. 검색 결과가 1건이거나 현재 페이지 끝에 도달한 경우입니다. 페이지네이션으로 다음 페이지로 이동 후 재진입하세요.

### Q. 이미지 저장 파일이 한글 폰트가 깨져 나옵니다.
A. 브라우저가 Pretendard를 로드 못 한 상태일 수 있습니다. 페이지를 새로고침 후 다시 시도하세요. 일반적으로 정상 동작합니다.

### Q. 주민번호 입력창에 원래 값이 보이지 않습니다.
A. 개인정보 보호를 위해 의도된 동작입니다. 저장된 값은 암호화·해시되어 복원되지 않으며, 관리자만 "복호화" 버튼으로 1회성 열람할 수 있습니다. 변경 시 전체 13자리를 다시 입력해야 합니다.
